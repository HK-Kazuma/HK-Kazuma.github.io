なぜ作ったか

チームでプロダクトを作ってみたいという思いから開発を始めた。社会問題を解決するプロダクトを作るのはハードルが高いため、身の回りの課題に着目。パソコンが使えない状況でメモ帳に書いたメモを、後からパソコンでまとめ直す作業が負担に感じていたことから、本プロダクトの開発に至った。

工夫した点

開発期間が2週間という制限があったため、チームメンバー全員が経験のあるPythonをベースに開発を進めた。手書き文字の認識には従来のOCR技術ではなく、最新のAIを活用したOCRを採用し、認識精度の向上を図った。

なぜ作ったか

研究テーマを決めるための情報収集と、最新技術に日常的に触れたいという2つの動機がある。AI関連の最新情報を得ることで、世界的な潮流を把握し、今求められていることや懸念点を抽出して自分の身の回りの課題と結びつけることで、解決すべきテーマを見出すことを目的とした。

工夫した点

目覚まし同盟

グループで起床を管理するアラームアプリ。起床確認後にアプリをブロックする機能を実装。 iOS / Android対応予定。

React Native Expo FastAPI Supabase Firebase 個人開発 開発中
↗

なぜ作ったか

朝起きられない原因のひとつに、他者からの監視がないことが挙げられる。実家では親が起こしに来るという要因が起床につながるが、一人暮らしでは起床が完全に自己管理となり、難しいと感じる人が多い。同じ時間に起きる人たちとつながり、互いに監視し合う状態を作ることで起床を促すアプリを開発している。

工夫した点

就寝時や起床時に手が届きやすいスマホで使うことを前提に開発。起床後にだらだらスマホを触ることを防ぐため、SNS系アプリを制限する機能を追加。さらに起床後のタスクを表示させることで、朝の時間を有効に使えるよう設計している。

「AIに教える」ことで理解を深めるフェインマン学習法アプリ。 LLMが知識と理解度を2段階で評価する仕組みを実装予定。

なぜ作ったか

生成AIの普及により思考力の低下が最新の研究で示されている。本来AIの活用は能力向上に貢献すべきだと考えており、その適切な活用方法とは「AIに考えることを丸投げしないこと」である。適切で良質な質問を作ることはAIには難しく、だからこそ質問する力・疑問を生み出す力を人間が鍛えるべきだと考えた。自ら問いを投げることが難しい人に向けて、AIが問いを出し思考を補助することで、思考力を維持・向上させることを目的としている。

工夫した点

勉強したい内容についてスムーズにやり取りできるよう、チャット形式の応答を採用。テキストの画像や資料をアップロードできる機能や、勉強内容ごとに分けて保管・管理できるユーザー作成ノート機能を実装予定。
